24. Scripting API Reference | Project.Model.Track

Types:

  • model.AudioTrack Base class for tracks that contain Sounds.
  • model.GroupTrack A track within an Event that is associated with a MixerGroup.
  • model.MarkerTrack A track containing markers.

Extension Methods:

  • AudioTrack.addAutomationTrack Creates an automation track for a given property of an automatable object.
  • GroupTrack.addSound Adds a sound to the track on the parameter specified.
  • GroupBus.getItem Gets the MixerBus at the relative path specified.
  • MarkerTrack.addNamedMarker Adds a named marker to the track at the given position.
  • MarkerTrack.addRegion Adds a region to the track at the given position with the given length.
  • MarkerTrack.addSustainPoint Adds a sustain point to the track at the given position.
  • MarkerTrack.addTransitionMarker Adds a transition marker to the track at the given position.
  • MarkerTrack.addTransitionRegion Adds a transition region to the track at the given position with the given length.

AudioTrack.addAutomationTrack

Creates an automation track for a given property of an automatable object.

AudioTrack.addAutomationTrack(
    automatableObject,
    propertyName
)
automatableObject
Any model.AutomatableObject of the track such as a model.Sound, a model.MixerEffect or the model.MixerBus of the track itself.
propertyName
Any valid automatable property of the specified model.AutomatableObject.

Returns the automation track managed object.

See Also: Tracks

For example:

// adds an automation track for a master track's volume
var event = studio.project.workspace.addEvent("New Event", true);
var masterTrack = event.masterTrack;
var automationTrack = masterTrack.addAutomationTrack(masterTrack.mixerGroup, "volume");

// adds an automation track for a gain effect's gain
var gainEffect = masterTrack.mixerGroup.effectChain.addEffect("GainEffect");
var automationTrack2 = masterTrack.addAutomationTrack(gainEffect, "gain");

model.AudioTrack

Base class for tracks that contain Sounds.

See Also: model.Sound

GroupTrack.addSound

Adds a sound to the track on the parameter specified.

GroupTrack.addSound(
    parameter,
    soundType,
    start,
    length
)
parameter
The Parameter Sheet to add the sound to.
soundType
The type of the model.Sound. Must be model.SingleSound, model.MultiSound, or model.ProgrammerSound.
start
Start position number of new model.Sound. Value must be in the range of the parameter.
length
number length of model.Sound to add.

Returns the model.Sound managed object.

See Also: Tracks, model.Parameter

GroupBus.getItem

Gets the MixerBus at the relative path specified.

GroupBus.getItem(
    path
)
path
Relative string path of the entity to get the model.MixerBus of.

Returns the model.MixerBus at the relative path specified. Returns undefined if the bus cannot be found.

Example:

var bus = studio.project.workspace.mixer.masterBus.getItem("sfx/player");

MarkerTrack.addNamedMarker

Adds a named marker to the track at the given position.

MarkerTrack.addNamedMarker(
    name,
    position
)
name
string name to assign the Marker.
position
number position to place Marker.

Returns the named Marker managed object.

MarkerTrack.addRegion

Adds a region to the track at the given position with the given length.

MarkerTrack.addRegion(
    position,
    length,
    name,
    loopMode
)
start
number Start position of new Region.
length
number length of Region to add.
name
string The name of the region.
loopMode
number representing the region's loop mode. Defaults to project.regionLoopMode.Looping if loopMode is undefined. The loop mode can be set by passing in a valid loopMode argument, as described by the Project.regionLoopMode enum.

Returns the Region managed object.

MarkerTrack.addSustainPoint

Adds a sustain point to the track at the given position.

MarkerTrack.addSustainPoint(
    position
)
position

Returns the SustainPoint managed object.

MarkerTrack.addTransitionMarker

MarkerTrack.addTransitionMarker(
